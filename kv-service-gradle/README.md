# KV gRPC Service
## Системные требования

- Java 11+
- Docker (для локального тестирования Tarantool)
- `./gradlew` (включён в проект)

1) Запустить Tarantool:

```bash
docker run -d --name tarantool -p 3301:3301 tarantool/tarantool:3.2
```

2) Собрать и проверить тесты:

```bash
cd kv-service-gradle
./gradlew clean test
```

3) Запустить сервис:

```bash
./gradlew run
```

## API 
- `Put(key, value)` сохраняет/обновляет
- `Get(key)` возвращает `optional bytes`
- `Delete(key)` удаляет
- `Range(key_since, key_to)` стримит по ключам
- `Count()` возвращает общее количество

## API Методы

### 1. Put(key, value)
Сохраняет или обновляет значение.
- **Параметры**: key (string), value (bytes, nullable)
- **Возвращает**: пустой ответ

### 2. Get(key)
Получает значение по ключу.
- **Параметры**: key (string)
- **Возвращает**: bytes (value) или null

### 3. Delete(key)
Удаляет запись по ключу.
- **Параметры**: key (string)
- **Возвращает**: пустой ответ

### 4. Range(key_since, key_to)
Получает значения в диапазоне ключей (stream).
- **Параметры**: key_since (string ≤ key_to)
- **Возвращает**: поток пар (key, value)

### 5. Count()
Возвращает количество записей.
- **Примечание**: эффективно для 5M+ записей
